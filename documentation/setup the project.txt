# Virtual Environment Setup & Service Launch Guide

This document provides step-by-step instructions for creating Python virtual environments, installing dependencies, and running the required services for the **Harare Chatbot** system.  
The setup includes four components:

- **Rasa** (chatbot engine + actions server)
- **RAG System** (retrieval-augmented generation API)
- **LLM Gateway** (language model proxy API)
- **Harare Chatbot Dashboard** (web UI)

All commands are written for **PowerShell** on Windows (since activation scripts use `.\venv\Scripts\Activate.ps1`).  
Adjust paths if your project structure differs.

---

## 1. Prerequisites

- **Python 3.8–3.11** installed (check with `python --version`)
- **pip** and **venv** module (included with Python)
- PowerShell execution policy may need adjustment:
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```

---

## 2. Project Directory Structure (assumed)

```
C:\YourProject\
├── .venv_rasa\
├── .venv_rag\
├── .venv_llm\
├── .venv_harare_chatbot_dashboard\
├── rasa\                         (contains requirements.txt)
├── rag_system\                   (contains app/ and requirements.txt)
├── llm_gateway\                  (contains app/ and requirements.txt)
└── harare_chatbot_dashboard\     (contains app.py & requirements.txt)
```

> **Important:** Each component folder must contain a `requirements.txt` file listing its dependencies.

---

## 3. Creating Virtual Environments

Open a PowerShell terminal **in the root project folder** (`C:\YourProject`).

### 3.1 Rasa Environment
```powershell
python -m venv .venv_rasa
```

### 3.2 RAG System Environment
```powershell
python -m venv .venv_rag
```

### 3.3 LLM Gateway Environment
```powershell
python -m venv .venv_llm
```

### 3.4 Dashboard Environment
```powershell
python -m venv .venv_harare_chatbot_dashboard
```

---

## 4. Installing Requirements

For each environment, **activate it first**, then install dependencies from the corresponding folder.

### 4.1 Rasa Dependencies
```powershell
.\.venv_rasa\Scripts\Activate.ps1
cd rasa
pip install -r requirements.txt
cd ..
```

### 4.2 RAG System Dependencies
```powershell
.\.venv_rag\Scripts\Activate.ps1
cd rag_system
pip install -r requirements.txt
cd ..
```

### 4.3 LLM Gateway Dependencies
```powershell
.\.venv_llm\Scripts\Activate.ps1
cd llm_gateway
pip install -r requirements.txt
cd ..
```

### 4.4 Dashboard Dependencies
```powershell
.\.venv_harare_chatbot_dashboard\Scripts\Activate.ps1
cd harare_chatbot_dashboard
pip install -r requirements.txt
cd ..
```

> **Tip:** Use `pip install --upgrade pip` inside each venv before installing requirements to avoid compatibility warnings.

---

## 5. Running the Services

Each service runs in its **own terminal window** (or as background jobs). Keep all terminals open while the chatbot is in use.

### 5.1 Rasa Actions Server (port 5055)

**Terminal 1**
```powershell
.\.venv_rasa\Scripts\Activate.ps1
cd rasa
rasa run actions --port 5055
```
Expected output: `Custom action server started on http://localhost:5055`

---

### 5.2 Rasa API Server (port 5005)

**Terminal 2**
```powershell
.\.venv_rasa\Scripts\Activate.ps1
cd rasa
rasa run --enable-api --cors "*" --port 5005
```
This exposes REST endpoints for the dashboard to communicate with Rasa.

---

### 5.3 RAG System API (port 8000)

**Terminal 3**
```powershell
.\.venv_rag\Scripts\Activate.ps1
cd rag_system
uvicorn app.main:app --port 8000
```
The RAG service provides context retrieval for the LLM.

---

### 5.4 LLM Gateway API (port 8001)

**Terminal 4**
```powershell
.\.venv_llm\Scripts\Activate.ps1
cd llm_gateway
uvicorn app.main:app --port 8001
```
This gateway handles requests to the actual language model.

---

### 5.5 Dashboard (default Flask port 5000)

**Terminal 5**
```powershell
.\.venv_harare_chatbot_dashboard\Scripts\Activate.ps1
cd harare_chatbot_dashboard
python app.py
```
Open a browser and go to `http://localhost:5000` to use the chatbot interface.

---

## 6. Verification Checklist

| Service               | URL / Command               | Expected Response                     |
|-----------------------|-----------------------------|---------------------------------------|
| Rasa Actions          | `http://localhost:5055`     | 404 or `{"error":"..."}` but server runs |
| Rasa API              | `http://localhost:5005`     | `{"status":"ok"}` or version info     |
| RAG System            | `http://localhost:8000/docs` | Swagger UI page                      |
| LLM Gateway           | `http://localhost:8001/docs` | Swagger UI page                      |
| Dashboard             | `http://localhost:5000`      | Web page with chat interface          |

## 7. Troubleshooting

### 7.1 “Activate.ps1 cannot be loaded because running scripts is disabled”
Run PowerShell as Administrator and set execution policy:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 7.2 “Module not found” after installing requirements
Make sure you are in the correct virtual environment (check prompt shows `(.venv_xxx)`).  
Reinstall: `pip install -r requirements.txt --force-reinstall`

### 7.3 Port already in use
Stop the process using that port or change the port in the command:
```powershell
rasa run --port 5006          # example
uvicorn app.main:app --port 8002
```

### 7.4 Rasa cannot find actions
Ensure `actions.py` exists in the `rasa` folder and the actions server is started **before** the main Rasa server.

---

## 8. Stopping Everything

- In each terminal press `Ctrl + C` and confirm with `y` if needed.
- Close the terminals or deactivate each environment with:
  ```powershell
  deactivate
  ```

---

## 9. Automation (Optional)

To launch all services with one script, create a PowerShell script (`start_all.ps1`) with the following:

```powershell
# Start Rasa Actions
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; .\.venv_rasa\Scripts\Activate.ps1; cd rasa; rasa run actions --port 5055"

# Start Rasa API
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; .\.venv_rasa\Scripts\Activate.ps1; cd rasa; rasa run --enable-api --cors '*' --port 5005"

# Start RAG
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; .\.venv_rag\Scripts\Activate.ps1; cd rag_system; uvicorn app.main:app --port 8000"

# Start LLM Gateway
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; .\.venv_llm\Scripts\Activate.ps1; cd llm_gateway; uvicorn app.main:app --port 8001"

# Start Dashboard
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; .\.venv_harare_chatbot_dashboard\Scripts\Activate.ps1; cd harare_chatbot_dashboard; python app.py"
```

Run it from the project root: `.\start_all.ps1`

---

## 10. Summary

You now have a complete environment for the Harare Chatbot. Each service runs in isolation, ensuring no dependency conflicts. Remember to:

- Create the virtual environments once.
- Install requirements after each environment is created.
- Run each service in its own terminal.
- Verify all ports are free before launching.

For updates to dependencies, re-run the `pip install -r requirements.txt` step inside the relevant venv.
